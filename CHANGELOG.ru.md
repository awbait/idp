## [0.1.0] - 2026-06-19

Первый тегированный релиз. Консоль платформы (portal) даёт командам
самообслуживание: разворачивать продукты из каталога чартов, GitOps-провижининг,
согласование публикаций чартов, OIDC-аутентификацию и наблюдаемость.

### Добавлено
- Портал самообслуживания: заказ и обновление продуктов из каталога,
  GitOps-провижининг в приложения Argo CD, уникальность идентичности ресурсов в
  рамках namespace.
- Публикации чартов: категории, владельцы и view-документ в базе, конечный
  автомат согласования (отправка, отзыв с согласования, отзыв публикации), смена
  категории и владельца через согласование.
- View-документы: динамические enum'ы и вычисляемые колонки, валидация
  документов, ровно одна каноническая order-view на чарт.
- Каталог: добавление чарта по произвольному пути в Harbor с проверкой
  комплектности, статус публикации на карточках, меню и каталог из публикаций.
- Конструктор: редактор view-документа и values.schema.json с реальным
  предпросмотром заказа/продукта и встроенной справкой по формату.
- Аутентификация: вход через OIDC/Keycloak, RP-initiated logout, возврат на
  страницу после повторного входа по 401, RBAC-роли (member, admin, support,
  security с разделом InfoSec).
- Наблюдаемость: метрики платформы с дашбордом Grafana и структурные логи с
  тегом компонента.
- Collector: снимает состояние Kubernetes в Valkey для консоли.
- Портал отдаёт встроенную SPA напрямую (nginx убран) и показывает страницу
  About с версией сборки и ссылками.
- Отдельные страницы документации.
- CI/релизный пайплайн на GitHub Actions: проверки PR, тег + GitHub Release при
  мерже `release/*` PR, сборка и публикация multi-arch образов portal и collector
  в GHCR по тегам `v*`.
