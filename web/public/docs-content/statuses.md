# Статусы и развёртывание

После отправки заказ проходит несколько этапов: согласование, оформление изменения в Git и развёртывание в кластере. Текущий этап виден в списке заказов и на странице заказа.

## Жизненный цикл заказа

| Статус | Что означает |
|--------|--------------|
| Черновик | Заказ создан, но ещё не отправлен |
| На согласовании | Заказ ждёт решения ответственного |
| Принят | Согласован, оформляется изменение в Git |
| Разворачивается | Платформа приводит кластер к заказанному состоянию |
| Готов | Сервис развёрнут и работает |
| Отклонён | Согласующий вернул заказ с замечаниями |

## Где смотреть развёртывание

Состояние развёртывания платформа получает из системы доставки (GitOps). На странице заказа отображается итоговое состояние сервиса: синхронизирован ли он с заказанной конфигурацией и здоров ли.

## Если что-то пошло не так

- **Заказ завис на согласовании** - напомните ответственному; согласующий виден в карточке заказа.
- **Развёртывание не доходит до «Готов»** - откройте страницу заказа: там отображается причина (например, ошибка проверки параметров).
- **Нужны изменения** - не создавайте новый заказ, а отредактируйте существующий.

> [!WARNING]
> Не меняйте конфигурацию сервиса в обход портала: GitOps вернёт состояние к тому, что описано в заказе.
