# Панель поддержки

Отдельный раздел портала для службы поддержки (роль support). Переключиться в него можно вверху бокового меню; у роли support он открывается по умолчанию.

Поддержка - кросс-командная роль: она видит и помогает с заказами **всех** команд, поэтому раздел не привязан к конкретной команде (селектор проекта в нём не показывается).

## Обзор

Сводная панель по всем заказам платформы в реальном времени:

- **Сервисов всего** - сколько экземпляров уже развёрнуто или разворачивается;
- **Работают** - сервисы в состоянии Healthy;
- **Требуют внимания** - сколько сервисов в проблемном состоянии (Degraded, нет в ArgoCD) или с расхождением с Git (дрейф);
- **В процессе** - заказы, которые сейчас разворачиваются или ждут слияния.

Ниже - список **«Требуют внимания»**: проблемные и разошедшиеся с Git сервисы с указанием команды и статуса. Клик по строке открывает заказ. Это рабочий список поддержки: с него удобно начинать разбор инцидентов, не переключаясь между командами.

## Заказы всех команд

Полный список заказов по всем командам с колонкой **Команда**. Список фильтруется:

- по **командам** - с полем поиска, чтобы быстро найти нужную;
- по **продуктам** (типам сервисов) - тоже с поиском;
- по **статусу** заказа.

Фильтры комбинируются, рядом есть кнопка сброса. Отсюда поддержка открывает любой заказ и при необходимости правит его.

## Что может поддержка

- открывать и просматривать любой заказ любой команды;
- изменять параметры заказа и обновлять версию сервиса;
- видеть историю действий, дрейф и запросы на слияние.

Поддержка **не** создаёт и **не** удаляет заказы и не участвует в согласованиях - это остаётся за командами-владельцами и администраторами. Подробнее о разграничении - в разделе [Роли и доступы](roles).
