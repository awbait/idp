# Роли и доступы

Доступ к порталу определяется вашей ролью. Роль одна на пользователя и вычисляется из ваших групп в Keycloak при входе. Если у вас несколько групп, выбирается старшая по приоритету:

**admin > support > security > member > auditor**

## Краткая сводка

| Роль | Что доступно |
|---|---|
| Участник (member) | Заказы своих команд: просмотр, создание, изменение, удаление. Добавление чартов в каталог и отправка формы на согласование. |
| Поддержка (support) | Просмотр и изменение заказов всех команд. Без создания и удаления, без согласований. |
| Информационная безопасность (security) | Раздел ИБ: согласование политик и доступ к Kyverno. Доступа к заказам команд нет. |
| Администратор платформы (admin) | Полный доступ: каталог, категории, согласование публикаций, статус системы. |
| Аудитор (auditor) | Роль по умолчанию, только просмотр. Выдаётся, если нет других групп. |

## Участник

Базовая роль для работы с сервисами. Участник действует от имени своей команды (группа `team-*`), а если команд несколько - выбирает активную в верхней панели.

- заказывает новые экземпляры сервисов и меняет уже заказанные;
- удаляет свои заказы;
- добавляет чарты в каталог и отправляет форму заказа на согласование.

Участник видит только заказы своих команд.

## Поддержка

Роль для сопровождения: поддержка видит и правит заказы **всех** команд, не привязана ни к одной из них.

- просмотр и изменение любого заказа (значения, обновление версии);
- список заказов и приложения по всем командам.

Поддержка **не** создаёт и **не** удаляет заказы и не участвует в согласованиях - это остаётся за участниками команд и администраторами.

## Информационная безопасность

Отдельный раздел портала для ИБ. Переключиться в него можно вверху бокового меню; у роли security он открывается по умолчанию. Команда у этой роли не учитывается - доступ даёт сама роль.

В разделе:

- **Обзор** - сводка по согласованиям и соответствию политикам;
- **Согласование политик** - очередь заказов сервиса policies всех команд;
- **Kyverno UI** - отчёты по политикам кластера.

> [!NOTE]
> Раздел ИБ сейчас в разработке: экраны показывают демонстрационные данные. Реальные согласование политик и интеграция с Kyverno появятся вместе с сервисом policies.

Доступа к обычным заказам команд у роли security нет.

## Администратор платформы

Полный доступ к платформе. В дополнение к возможностям участника администратор:

- управляет каталогом и категориями;
- согласует публикации (формы заказа управляемых сервисов);
- видит статус системы (интеграции и хранилища);
- видит разделы и Платформы, и ИБ.

## Аудитор

Роль по умолчанию: выдаётся, если у пользователя нет ни команды, ни специальной группы. Доступ ограничен просмотром общедоступных разделов (каталог, документация); заказы команд аудитору не видны.

## Как назначается роль

- Роль выводится из ваших групп в Keycloak при каждом входе.
- Команды - это группы с префиксом `team-` (например `team-core`); суффикс - имя команды/проекта.
- Группы для ролей admin, support и security настраиваются платформой.
- Роли support и security - платформенные: членство в командах для них не учитывается.

> [!TIP]
> Не хватает доступа или нужна другая роль? Обратитесь к администратору платформы - роли и команды назначаются через группы Keycloak.
