# Структура сервисов в Git

Все управляемые сервисы хранятся в одной GitLab-группе **managed-services**. Каждый ваш заказ Console превращает в файлы в этой структуре, а Argo CD разворачивает их в кластере. Знать её устройство полезно, чтобы понимать, что стоит за заказом, и ориентироваться в ссылках на merge request.

> [!NOTE]
> Эти файлы создаёт и обновляет сам портал. Вручную их обычно не трогают: правка в обход портала приводит к расхождению (дрейфу). См. раздел [Статусы и развёртывание](statuses).

## Иерархия

Внутри группы - три уровня: команда -> сервис -> экземпляр. Конфигурация каждого экземпляра лежит в Git как пара YAML-файлов.

```
managed-services/                       группа managed-services
├── <team>/                             подгруппа команды
│   ├── ingress-gateway/                репозиторий = управляемый сервис (чарт)
│   │   └── <cluster>/                  кластер
│   │       └── <namespace>-igw-<gw>/   папка экземпляра
│   │           ├── application.yaml
│   │           └── values.yaml
│   └── postgres/
│       └── <cluster>/
│           └── <service-name>/
│               ├── application.yaml
│               └── values.yaml
└── <team>/
    └── ...
```

## Уровни

- **Подгруппа команды** - на каждую команду своя подгруппа, имя совпадает с названием команды.
- **Репозиторий сервиса** - внутри подгруппы по репозиторию на каждый управляемый сервис; имя репозитория совпадает с именем сервиса (чарта), например `ingress-gateway`.
- **Папка экземпляра** - внутри репозитория отдельная папка на каждый экземпляр, сгруппированная по кластеру: `<cluster>/<service-name>`. Группировка по кластеру разделяет экземпляры одного сервиса в разных кластерах.

## Файлы экземпляра

В папке экземпляра два файла:

| Файл | Содержимое |
|------|------------|
| `application.yaml` | Манифест Argo CD `Application` - описывает развёртывание экземпляра |
| `values.yaml` | Значения Helm-чарта (параметры заказа) для этого экземпляра |

`application.yaml` ссылается на сам чарт (берётся из реестра Harbor с зафиксированной версией) и на соседний `values.yaml`. Так манифест полностью описывает развёртывание, а параметры экземпляра остаются в Git. `values.yaml` должен соответствовать схеме чарта - именно его проверяет форма заказа.

## Соглашения об именовании

| Уровень | Шаблон | Пример |
|---------|--------|--------|
| Группа | `managed-services` | `managed-services` |
| Подгруппа команды | `<team>` | `team-core` |
| Репозиторий | `<service>` | `ingress-gateway` |
| Папка экземпляра | `<cluster>/<service-name>` | `prod/payments-igw-public` |
| Файлы | `application.yaml`, `values.yaml` | - |
