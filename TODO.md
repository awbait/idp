## 3. Интеграция с NetBox (IPAM)

NetBox - источник правды по IPAM. Консоль должна ходить в NetBox по API при
провижининге и выделять адресные ресурсы (новый апстрим-клиент `internal/netbox`
рядом с Harbor/GitLab/ArgoCD; real-клиент + fake для тестов).

- [ ] **Выделять subnet.** При заведении команды/namespace выделять подсеть
  (prefix) из родительского пула NetBox. Выданный CIDR прокидывать в провижининг
  (namespace-чарт уже рендерит `subnet.yaml` по этому значению).
- [ ] **Выделять VIP в заданной subnet.** В рамках выделенной подсети резервить
  VIP (IP-адрес) под сервисы, которым он нужен (например ingress-gateway / LB).
- [ ] Конфиг: `NETBOX_URL` + `NETBOX_TOKEN` (в `config.go` и `.env.example`),
  родительский пул/роль префикса - параметрами. Освобождать ресурсы при удалении
  заказа/namespace, чтобы не плодить «висящие» аллокации.
- [ ] Логи и метрики на вызовы NetBox (по правилам наблюдаемости): исход

## 6. Код-ревью (остаток)

- [ ] **FE-M2 (хвост).** Стабильные ключи для `ArrayField` (`SchemaForm.tsx:600,643`) и
  `ListEditor` (`GenericProductTabs.tsx:288`). Сейчас `key={i}` (индекс): при
  удалении из середины раскрытое состояние `Disclosure`, фокус и значения
  «съезжают» на соседний элемент. `MapField` уже исправлен (стабильный `id`); для
  массивов элементы выводятся из `value` без локальной идентичности, поэтому нужен
  uid в lockstep со всеми мутациями (add/remove/reorder/setAt) + ресинк по длине
  при внешнем изменении `value`. Требует ручной проверки в SPA.

Принятые решения (не делаем):

- **L4** (рост in-memory cache) - memory-cache только в тестах/локально; в проде Redis с TTL.
- **L7** (N+1 `ListMRs`/`ListEvents`) - один `request_id` за вызов на детальной странице, не горячий путь.
- **L14** (rate limiting) - делается на уровне гейтвеев перед порталом, не в приложении.
